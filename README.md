Постановка задачи
Разработать рекомендательную систему, способную выдавать персонализированные рекомендации пользователям в реальном времени. 
Формат входных и выходных данных
Входные данные:  все комментарии конкретного пользователя на товары, поля - reviewID, nickname, ProductName, TextReview, Style, asin. Выходные данные: информативные эмбеддинги для пользователей, собранные на основе их фидбека и комментариев, и эмбеддинги для предоставляемых продуктов.   
Метрики
F1-score, Recall@K, acurracy по каждому продукту, Precision@K, особенно важна recall, поскольку при рекомендации мы хотим в первую рекомендовать тем, кому это правда может быть интересно.
Валидация и тест
Для разделения выборки используется случайное разбиение: из исходного train-набора выделяется 5% данных для валидации с помощью функции random_split из torch.utils.data.dataset. Для контроля ключевых метрик - F1-score, recall, accuracy, precision, ROC-AUC будет использован ml-flow.
Датасеты
Взят датасет Amazon Review Data (https://nijianmo.github.io/amazonl) в разделе musical instruments, он представляет из себя 231392 json объектов, которые являются отзывами различных пользователей на товары, каждый из объектов содержит -  reviewID, nickname, ProductName, TextReview, Style, а также уникальный идентификатор asin, который может быть использован для сбора дополнительной информации о продукте.
